The package now is organized into a simple and practical directory
structure to help you easily follow the workflow from data preparation
to result analysis. Each folder and script has a specific role in the
inversion process.

<img src="../images/00_img1.jpg" style="width:6.5in;height:2.52153in" />

**Main directory**

- **SetupData**:
  This folder contains everything related to preparing the input data.
  It includes scripts and files needed to organize raw data into the
  correct format for the inversion.

- **Codes**:
  This is the core directory where the main inversion programs are
  stored. All scripts that perform the MCMC inversion are located here.

- **Project directory**:
  Central workspace where all data, input models, inversion setting files,
  and output results are stored.

- **AnalyzeResult**:
  After running the inversion, the output results are processed and
  visualized using the tools in this folder. It helps you understand and
  interpret the inversion results.

**Workflow scripts** (placed in <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">{Main direcotry}</span>)

All the works will be called by those script. The workflow is controlled by 4 main scripts:

- <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">**01_prepare_data.sh**</span>:
  This script prepares and organizes the input data. It ensures that all
  required files are in the correct format and location before running
  the inversion.

- <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">**02_do_MCMC.sh**</span>:
  This script runs the MCMC inversion. It uses the prepared data and
  model settings to generate a set of possible solutions.

- <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">**03_re_prepare_data.sh**</span>:
  This script is intended for re-preparing the data (e.g., changing the
  weighting or removing a data type for testing). However, it is rarely
  used, since running 
  <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">01_prepare_data.sh</span> 
  and 
  <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">**02_do_MCMC.sh**</span> is
  usually a better approach.

- <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">**04_re_do_MCMC_post_process.sh**</span>:
  Performs post-processing of the MCMC inversion when needed. It reads
  the posterior distributions from binary files and re-applies the
  parameter search ranges to select a refined subset of models.

- <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">**05_do_replot_MCMC.sh**</span>:
  Regenerates the output plots of the inversion results. This script is
  useful when adjusting visualization styles, such as switching between
  layer-cake and gradient representations.

**Project directory**

- **data**:
  This folder contains the observed data used for inversion (e.g.,
  dispersion curves or waveform-derived measurements). These are the
  inputs that the model tries to fit. The data was stored in the
  subdirectories as **{Project}/data/{sta}\_data/\***

- **Vel_mod**:
  This folder stores the velocity model settings or initial/reference
  models used during the inversion process.

- **MonterCarlo**:
  This folder stores both the initial setup and the output generated
  after the inversion.

- **Station files:** (placed in <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">{Project direcotry}</span>)

<div style="margin-left: 40px;">

<p><strong><code><span style="color:red;">station_cor.lst </span></code></strong><br>
Station list in the format: <em>name – longitude – latitude – elevation</em>.<br>
For the MCMC workflow, only the station name is required; the other parameters are used for plotting purposes only. This file is used during the data setup stage.
</p>

<p><strong><code><span style="color:red;">sta_now</span></code></strong><br>
Defines the stations to be processed in the MCMC inversion. It can include all stations from <code><span style="color:red; font-family:'Courier New', monospace; font-weight:bold;"> station_cor.lst</span></code> or a selected subset (e.g., a single station).
</p>

</div>

<h3>Summary</h3>

<p>The overall workflow follows a simple sequence:</p>

<ol>
  <li>
    Prepare data → 
    <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">
      SetupData + 01_prepare_data.sh
    </span>
  </li>

  <li>
    Run inversion → 
    <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">
      Codes + 02_do_MCMC.sh
    </span>
  </li>

  <li>
    Analyze results → 
    <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">
      AnalyzeResult (02_do_MCMC.sh)
    </span>
  </li>

  <li>
    Re-do the post-process analysis if needed
    (
    <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">
      04_re_do_MCMC_post_process.sh
    </span>
    and
    <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">
      05_do_replot_MCMC.sh
    </span>
    )
  </li>
</ol>

</div>

This structure is designed to keep the process **clear, modular, and
easy to debug**, especially for new users.
