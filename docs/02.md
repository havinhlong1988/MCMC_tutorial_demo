<ul style="text-align: justify; margin-left: 0; padding-left: 0px;">
During this step, model parameterization is defined based on the input
1D velocity model and is automatically handled by the scripts in the
<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">SetupData</span> folder. You still need to manually define some key
parameters. The script will read the input model and convert it into the
parameterized form required for the MCMC inversion.
</ul>

<ul style="text-align: justify; margin-left: 0; padding-left: 0px;">
The script also prepares and links all necessary files, including the
model file (<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">mod.{sta}</span>), parameter file (<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">in.para_{sta}</span>), data

(<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">in.connector</span>). This ensures that the inversion code can correctly
read and use all required inputs. This step is important because it
defines the model space that the MCMC algorithm will explore. All
parameters can be mainly set in the (<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">SetupData/parameter.py</span>) file.
</ul>

**1.** <strong>Convert 1D model to </strong><span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">mod.{sta}</span>:

To convert the 1D model into the input required for the MCMC workflow,
several parameters need to be considered, including:

<ul style="margin-left: 20px; text-align: justify;">
  <li>
  <span style="color:blue;"><strong>Mod_type_flag</strong></span>: choose the model input style.
  If = 1, use gradient style; if = 2, use B-spline style to smooth the
  model.
  </li>
</ul>

<ul style="margin-left: 20px; text-align: justify;">
  <li>
  <span style="color:blue;"><strong>Vel_mod_uniform_flag</strong></span>: choose whether to use one model for all
  stations or assign a separate model for each station.
  If = 1, use model name define later in (<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">{mod_file}</span>) for all; if = 0, use
all model in <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">Vel_mod</span>  direcotory under
<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;"><Vel_mod>{mod_file}/{sta}</span> .
  </li>
</ul>

<ul style="margin-left: 20px; text-align: justify;">
  <li>
  <span style="color:blue;"><strong>sed_flag, man_flag</strong></span>: setup the model with sedimentary layers and/or
  mantle layers (0 for no, 1 for yes).
  </li>
</ul>

<ul style="margin-left: 20px; text-align: justify;">
  <li>
  <span style="color:blue;"><strong>sed_value</strong></span>: when **sed_flag** = 1, use **sed_value** to define the
  initial values of the sedimentary layer in the input model.
  </li>
</ul>

<ul style="margin-left: 20px; text-align: justify;">
  <li>
  <span style="color:blue;"><strong>sednpara, crustnpara, mantlenpara</strong></span>: when **mod_type_flag = 2** ,
  these parameters are active and used to define the number of control
  points for each group of layers.
  </li>
</ul>

<ul style="margin-left: 20px; text-align: justify;">
  <li>
  <span style="color:blue;"><strong>sublay_thickchange</strong></span>: control parameter for sublayers variation
  (mainly focus on crustal group).
  </li>
</ul>

Other parameters also need to be taken into consideration (**scroll-down
to find them**):

<ul style="margin-left: 20px; text-align: justify;">
  <li>
    <span style="color:blue;"><strong>dds, ddc, ddm</strong></span>: define the sub-layer thicknesses for the sedimentary, crustal, and mantle groups. Regardless of how many sub-layers are defined in the input 1D model, these parameters control how the model is parameterized during inversion. Note that the current setting supports a maximum of 100 sub-layers across all three groups combined.
  </li>
</ul>

→ After running the script <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">01_prepare_data.sh</span>, the model for each station will be generated in <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">{project}/data/{sta}_data/mod.{sta}</span> with the proper value structure.

The parameter values are arranged following the logic below:

**Column Information:**

<span style="color:blue;"><strong>Column 1</strong></span>: perturbation type flag: (0 = values to be perturbed), (1 =
thickness to be perturbed)

<span style="color:blue;"><strong>Column 2</strong></span>: perturbation style flag: (1 = absolute perturbation), -1 =
relative perturbation (in %)

<span style="color:blue;"><strong>Column 3</strong></span>: perturbation range

<span style="color:blue;"><strong>Column 4</strong></span>: gaussian step width

<span style="color:blue;"><strong>Column 5</strong></span>: group layer id (consitent with group id in <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">mod.{sta}</span> file)

<span style="color:blue;"><strong>Column 6</strong></span>: sublayer id (consistent with number of value each group in
<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">mod.{sta}</span> file)

<span style="color:blue;"><strong><u>Note: All the ids count from 0</u></strong></span>

**Row Information:**

<span style="color:blue;"><strong>Rows group 0</strong></span>: (if sediment layers are set) : Sediment layer Vs
perturbation range (min and max range of velocity)

<span style="color:blue;"><strong>Rows group 1</strong></span>: Crustal layer Vs perturbation range (min and max range
of velocity/bspline)

<span style="color:blue;"><strong>Rows group 2</strong></span>: (if mantle layers are set) : Manle layer Vs
perturbation range (min and max range of velocity/bspline)

<span style="color:blue;"><strong>Rows group 3</strong></span>: (if sediment layers are set) : total sediment thickness
pertubation

<span style="color:blue;"><strong>Rows group 4</strong></span>: (if **crusl and mantle** layers are set): total crustal
thickness pertubation

<span style="color:blue;"><strong>Rows group 5</strong></span>: (if set) : Sediment layer thickness perturbation range
(top & bottom thickness range of sublayers)

<span style="color:blue;"><strong>Rows group 6</strong></span>: Crustal layer Vs perturbation range (top & bottom
thickness range of sublayers)

<span style="color:blue;"><strong>Rows group 7</strong></span>: (if set) : Manle layer Vs perturbation range (top &
bottom thickness range of sublayers)

**4. bridge file:** <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">in.connector</span>

<p style="text-align: justify;">
This file is used to define parameters mainly for forward modeling of receiver functions, as well as MCMC jump settings, multi-core processing, and post-run processing. All parameters can be modified in the script 
<code style="color:red; font-family:'Courier New', monospace; font-weight:bold;">Setupdata/parameter.py</code> 
using the following parameters:
</p>

<p style="text-align: justify;">
<span style="color:blue;"><strong>MC_inversion_type</strong></span>: set to <span style="color:blue;"><strong>1</strong></span> for gradient style or <span style="color:blue;"><strong>2</strong></span> for B-spline
style. Gradient style perturbs both velocity values and sublayer
thickness values, while B-spline style uses quadratic B-splines to
smooth the model. The B-spline style only provides variation in total
group thickness; sublayer variations are smoothed and therefore not
explicitly resolved.
</p>

<span style="color:blue;"><strong>RF_gaussian_width</strong></span>: input Gaussian factor used for forward modeling
of receiver functions. This values need to consistent with gaussian
factor that calculate the input receiver funtions.

<span style="color:blue;"><strong>MC_number_of_jump</strong></span>: number of jump to change the dimension on posterior distribution.

<span style="color:blue;"><strong>MC_number_of_iteration</strong></span>: defines how many samples are drawn at each jump.

<span style="color:blue;"><strong>MC_number_of_cores</strong></span>: defines how many cores use to run.

<span style="color:blue;"><strong>MC_percentage_post_process_select</strong></span>: defines the percentage of posterior models used to obtain the final model.

<span style="color:blue;"><strong>Depth_step</strong></span>: grid step for final model (in km).

<span style="color:blue;"><strong>modout_plot_type</strong></span>: define the plot style as layer-cake (<span style="color:blue;"><strong>0</strong></span>) or gradient (<span style="color:blue;"><strong>1</strong></span>).

<span style="color:blue;"><strong>MC_QC_thres</strong></span>: use the model quality control (<span style="color:blue;"><strong>1</strong></span>) or not (<span style="color:blue;"><strong>0</strong></span>). The quality control value set at the last row of file <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">{Project}/data/{sta}_data_/in.connector</span> the function name “<span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">goodmodel</span>” on <span style="color:red; font-family:'Courier New', monospace; font-weight:bold;">{Project}/Codes/MCMC_flex/CALmodel.C</span>
